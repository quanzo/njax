# Правила

* Документируй код: все классы должны иметь заголовок, описание работы и пример использования; все методы и функции должны иметь заголовок, описание функционала, и документированные в phpdoc параметры.
* Документацию и комментарии ведем на русском языке!
* Не используем структуры в виде массивов - для структурированной информации создаем DTO.
* `IArrayable` + `toArray()` — доменное преобразование DTO в PHP-массив (элементы могут быть объектами).
* `IJsonArrayable extends IArrayable` + `toJsonArray()` — сериализация DTO для `json_encode`; рекурсивная обработка — через `JsonArrayableHelper`.
* Для JSON-DTO, где `toArray()` совпадает с JSON, используй trait `ArrayableFromJsonTrait`.
* Метод `toScalarArray()` не используем — для JSON только `toJsonArray()`.
* Если у DTO разная семантика (`TaskIdCollectionDto`: `toArray()` → объекты, `toJsonArray()` → строки) — оба метода реализуют явно.
* Сеттеры оформляем во fluent стиле.
* Если есть повторяющийся функционал и он не привязан к объекту, то вынести его в хелпер.
* Избегай дублирование кода и создавай код пригодный к повторному использованию в проекте
* При написании тестов __обязательно__ должны проверяться граничные условия! Также в тестовых наборах данных должны быть заведомо неправильные данные для проверки реакции проверяемого кода! Минимальный набор тестовых данных - 10! Требуется комментировать каждый тест!
* При тестировании чего либо связанного с bash используй __БЕЗОПАСНЫЕ__ команды!
* В конце сессии, после внесения изменений запусти линтер, phpstan и тесты.
* Для проверки стиля PHP-кода используй `./vendor/bin/phpcs` (конфиг `phpcs.xml.dist`, базовый стандарт PSR-12, анализируются `src` и `tests`).
* после внесения изменений в проект __обязательно__ обновить информация в папке `docs/` Учтивай то, что файлы в директории `docs/` будут использоваться тобой как справочник. Там должны быть данные, которые упростят тебе работу!
* при создании/изменении тестов проекта __обязательно__ размести краткое описание в `docs/tests-overview.md`
* избегай shallow copy объектов.
* используй документацию `docs/` чтобы ориентироваться в коде проекта!

## Каталоги

* `src/` - исходный код
* `src/commands` - прикладные команды task-endpoint (`EchoTaskCommand`, `SumTaskCommand`, namespace `app\njax\commands`)
* `src/classes/commands` - базовый класс и реестр команд (`AbstractTaskCommand`, `TaskCommandRegistry`, namespace `app\njax\classes\commands`)
* `src/classes/dto` - DTO (подкаталоги: `http/`, `security/`, `task/request/`, `task/response/`, `task/queue/`)
* `src/classes` - корневой каталог классов (task, adapters, providers)
* `src/enums` - перечисления enums
* `src/helpers` - хелперы здесь храним
* `src/interfaces` - интерфейсы (подкаталоги: `serialization/`, `security/`, `task/command|executor|queue|retention|taskid/`)
* `src/classes/providers` - stub-реализации провайдеров (подкаталоги: `authorization/`, `executor/`, `queue/`, `retention/`, `signature/`, `taskid/`)
* `src/exceptions` - исключения (подкаталоги: `task/`, `security/`)
* `src/traits` - трейты (подкаталоги: `serialization/`, `collection/`)
* `docs/` - директория со справочной информацией
* `docs/tests-overview.md` - справочник всех PHPUnit-тестов (навигация по сценариям)
